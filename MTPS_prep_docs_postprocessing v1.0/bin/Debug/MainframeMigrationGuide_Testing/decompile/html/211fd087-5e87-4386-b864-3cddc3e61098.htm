Introduction to Application Modernization Assessment

Changing technology in order to support your new business requirements or to maximize efficiency requires a fundamental first step, namely, a review of what currently exists in your IT environment. Such a review is necessary if your enterprise is to make informed and correct decisions about the future of your IT assets.

Application Modernization Assessment (AMA) is such a process for understanding the existing environment, typically the mainframe environment, and recommending a modernization strategy based on a detailed analysis of the applications, especially COBOL applications, in this environment. It identifies the changes that will have to be made to source code during migration and produces a high-level modernization plan that outlines the resources, cost, and duration of migrating the applications to the new platform.

The Micro Focus AMA process recommends one of three modernization strategies, each of which is offered by Micro Focus:

  • Application Migration and Transformation. This option, also called Application Migration, offers enterprises the ability to drastically reduce the total cost of ownership (TCO) that is associated with key applications by moving them to a lower-cost platform for execution. Application Migration projects are significantly low risk and more cost effective than projects to re-write applications or to implement packaged applications.

  • zSeries Application Modernization. This option allows enterprises to re-use people, processes, and technology in order to extend the capabilities of existing mainframe applications with lowered risk and enhanced flexibility. This solution targets enterprises that have decided not to move to a new platform because they are satisfied with the value they are getting from their investment in their existing mainframe-based mission-critical systems. The zSeries Application Modernization option results in a more efficient development process that maximizes the value that enterprises derive from these existing systems.

  • COBOL technology. A little-known fact about the IT market is that some enterprises already use production-level COBOL applications on non-mainframe platforms. Such enterprises do not find either the zSeries Modernization option or the Application Migration option relevant to the modernization needs of their IT environments. Micro Focus has been addressing these needs for decades by providing COBOL solutions for distributed systems (for example, Microsoft Windows, UNIX, and Linux).

The AMA process uses an array of Micro Focus technologies that collect and analyze source code. These technologies produce a variety of reports that form the basis for deciding which of the above modernization strategies represents the optimal solution for the enterprise.

Tools-based Portfolio Approach

The Micro Focus AMA process adopts an automated tools-based approach to assessing an enterprise’s IT environment. Using automated tools is an efficient way to assess a mainframe-based IT environment because such an environment typically consists of applications that differ in their respective value and cost to the enterprise.

The use of tools developed by Micro Focus facilitates a high level of automated re-engineering—approximately 95 percent in application migration projects—that greatly reduces manual modifications of the applications and therefore reduces the risk of errors or omissions during the migration project. The use of these tools allows Micro Focus to repeat its assessment process with minimal effort and cost.

The core of the AMA process consists of the following:

  • Granular analysis. The unit of analysis can be a program statement, program, Job Control Language (JCL) job, or any other component in the application environment that is relevant to the migration or modernization project task. Each item in the workload is assigned a category that indicates the type of issues the item raises; the item is then stored in a central database to facilitate the analysis process.

  • Categories of issues. A category represents a type of migration issue or an action item. Each category is associated with a coding effort level and represents the time, in hours, that is required to resolve one instance of that kind of issue. Each category also has an associated testing effort, in hours, that represents the time that is required to test or verify the resolution of an issue. You can change the default settings for these effort levels to reflect the profile of your applications, type of work required, or other characteristics of your IT environment.

  • Analysis technology. The assessment analysis tool, Revolve Enterprise Edition, tallies the number of issues in each category. It searches for the effort levels that are associated with each category and arrives at a total for the coding and testing effort for the application. For example, in the case of migration projects, Micro Focus provides a specific set of migration assessment reports to provide information about a variety of metrics, including:

    • Total estimated migration effort by category, component, and the type of object—for example, program, JCL job, and data file.

    • Specific lines of code that will have to be changed and the reasons for changing them.

    • Identification of application content, exposing risk factors such as use of subsystems that may be unknown but would add complexity to the migration.

    • Potential migration issues in the code, such as code-level changes that may be required during a migration to Microsoft Windows and SQL Server™.

    • A rating of migration complexity, per program.

These reports enable the project team to rapidly assess the migration effort that will be required. The reports are supplemented by a comprehensive set of linked documentation that identifies and describes the application on a technical level.

Tools and Techniques

Micro Focus uses a variety of tools and techniques in its modernization assessment projects. The most important of these tools is Revolve Enterprise Edition 7.0.

Revolve Enterprise Edition 7.0

Revolve Enterprise Edition 7.0 is the Micro Focus flagship tool for assessing applications. It documents, analyzes, and understands mission-critical COBOL applications and their supporting infrastructures, and it identifies issues and risks that are involved in migrating these applications to a new environment.

Revolve also supports SOA and regulatory compliance initiatives by enabling enterprises to identify and expose business processes that are buried deep inside old systems that may have been developed decades earlier.

Revolve delivers reports in either hard copy or via a secured Web site. These reports include documentation about application source code, impact analysis, and the overall understanding of the entire makeup of core applications.

Revolve has other capabilities that may be relevant depending on the specific requirements of the project. For example, a large-scale modernization project will benefit from the tool’s built-in ability to search the knowledge repository by using such specific monetary data items as search criteria, which is a common requirement in many enterprises.

Revolve is available to enterprises directly from Micro Focus and from a wide range of Micro Focus partners who use Revolve analysis to estimate the time and effort that is required for consulting and outsourcing projects. Revolve can be automated as a repeatable process for ongoing system maintenance and continual refinement of your IT environment’s knowledge repository.

Other Analysis Tools

Revolve provides additional tools that complement its analysis capabilities. Many of these tools are specifically enabled to support migration projects, including:

  • Migration Support: Export Source Modules. Sends a logically complete subset of an application to other Micro Focus development tools such as Mainframe Express Enterprise Edition or Net Express.

  • Migration Support: SQL Statements. Examines SQL statements to determine what changes might be required. Where cursors are involved, it groups together the DECLARE, OPEN, FETCH, and CLOSE statements that correspond to each cursor.

  • Migration Support: Determine Run Unit. Assembles the components that are required to replicate a complete execution of a particular process (run unit), starting either from a JCL—including all procedures, control cards, and called programs—or from a CICS/IMS transaction through its associated downstream programs, imports, and screens.

Effort Estimation Template

The Effort Estimation Template is a Microsoft Excel workbook that contains the results of the discovery and analysis process. After the assessment analysis is completed, the detailed results are exported from Revolve to an Excel template so that the project team can use the features of Excel to adjust the project estimates. Revolve works with Excel to identify, categorize, and document the points of change that are required. The results of the assessment analysis are created by a batch process that executes the following three analysis tools:

  • Migration Extract Effort. Extracts the Enterprise Edition worksheet to a flat file that will be used to populate the data sheet of the estimation Excel workbook.

  • Migration Extract: Project Issue Categories. Extracts the category names from the worksheet to a flat file, which is used by the Update Category List analysis tool.

  • Migration Extract: Update Category List. Populates the Categories worksheet in the Effort Estimation workbook.

Implementing a Modernization Assessment Project

Micro Focus uses a mature methodology to plan and execute its modernization assessment projects.

Project Scope

Micro Focus offers a ten-day assessment consulting service called QuickStart Pack for enterprises that are preparing to migrate their existing COBOL applications to Microsoft Windows, UNIX, or the Web. QuickStart Pack has the following scope:

  • Determine objectives. Determine and document your goals and objectives in modernizing your COBOL-based applications.

  • Determine requirements. Determine and document your requirements in modernizing your COBOL-based applications.

  • Assess applications. Assess and document the current COBOL-based applications to determine their various components and their characteristics—for example, programs, screens, files, and complexity.

  • Document the solution. Document the recommended solution to achieve your pre-defined goals, objectives, and requirements.

  • Prepare project plan. Prepare a high-level project plan and task list, including estimated resource requirements, costs, and time frame of the project.

Project Deliverables

The assessment project produces several types of deliverables:

  • Web documents. A separate and comprehensive intranet Web site will hold all project documents as linked HTML pages that describe how the application components connect to each other. It provides a categorized list of the potential migration issues discovered during the assessment process and supports navigating to the component and issue level.

  • Worksheet. A Revolve Enterprise Edition worksheet that enables you to focus on the specific details of each potential issue in the Revolve toolset for further analysis. These details can be exported to an Excel spreadsheet to generate accurate estimates of the effort that is required to resolve the issue.

  • Project summary. This document is produced upon completion of the project.

  • Other deliverables. Additional deliverables and project materials can be produced depending on the type of the project type and depending on your additional requirements.
